Démarrage rapide

L'application fonctionne depuis un seul fichier index.html à ouvrir dans un navigateur (Chrome recommandé). Aucune installation, aucun compte en ligne requis.

Étape 1 — Ouvrir l'application

Double-cliquez sur index.html. Sur smartphone, dans Chrome, touchez le menu ⋮ → Ajouter à l'écran d'accueil pour une expérience PWA complète.

Compatibilité
Chrome 90+ et Edge 90+ — fonctionnement complet (scanner code-barres, GPS, météo). Firefox et Safari — fonctionnement sans scanner vidéo (import image disponible en remplacement).

Étape 2 — Assistant de configuration

Lancé automatiquement à la première ouverture. Il se déroule en 4 étapes :

ÉtapeContenuObligatoire
1 — ProfilPrénom, localisation, descriptionNon — peut être complété plus tard
2 — Mot de passeMinimum 6 caractères · haché SHA-256Oui
3 — GitHubToken + dépôt pour héberger les photosNon — peut être configuré plus tard
4 — ConfirmationRécapitulatif avant ouverture—
Mot de passe oublié
Depuis l'écran de connexion, le lien Mot de passe oublié réinitialise la configuration de sécurité (hash + token GitHub) mais conserve intégralement votre collection.

Étape 3 — Autoriser la géolocalisation

À la première ouverture du tableau de bord, le navigateur demande l'accès à votre position. Acceptez pour activer la météo automatique et le centrage GPS de la vue Verger. Vous pouvez saisir une ville manuellement si vous refusez.

Aide contextuelle — bouton ❓

Un petit bouton ? apparaît à côté des titres de chaque module (jauges d'arrosage, météo, verger, calendrier…). Touchez-le pour afficher un panneau d'aide rapide directement dans l'application, sans quitter l'écran. Un lien en bas du panneau ouvre la section correspondante de ce guide complet.

Étape 4 — Explorer et personnaliser

8 sujets sont pré-chargés avec leur nomenclature scientifique. Modifiez-les selon votre collection réelle, supprimez ceux que vous n'avez pas, et ajoutez les vôtres via le bouton ＋ de l'onglet Collection.

GitHub — configuration facultative

L'hébergement des photos sur GitHub est entièrement facultatif. Lors de l'assistant (étape 2), touchez → Passer cette étape pour ignorer. L'application fonctionne pleinement sans GitHub — les photos ne seront simplement pas disponibles. Pour configurer plus tard : Profil → 🔒 Sécurité → Modifier token / dépôt GitHub.

Créer un token GitHub
github.com → Settings → Developer settings → Personal access tokens → Fine-grained tokens. Accordez uniquement Contents : Read & Write sur le dépôt concerné. Le token est vérifié lors de la configuration.

E-mail de récupération du mot de passe

Saisissez un e-mail à l'étape 1 de l'assistant (ou dans Profil → 🌿 Profil → E-mail de récupération). Cet e-mail joue le rôle de question de sécurité locale — aucun message n'est jamais envoyé.

En cas de mot de passe oublié, touchez le lien Mot de passe oublié ? sous le formulaire de connexion. Le formulaire de récupération s'affiche directement dans la carte de connexion — accessible même lorsqu'on n'est pas connecté. L'adresse est affichée masquée (je***@example.com) et vous devez la saisir en entier pour autoriser la réinitialisation. Bouton ← Retour à la connexion pour annuler. Si aucun e-mail n'est renseigné, la réinitialisation reste possible mais sans vérification d'identité (un avertissement vous en informe).

Ce que la réinitialisation préserve
Vos données de collection (plantes, événements, historiques, coordonnées GPS) sont intégralement conservées. Seule la configuration de sécurité (mot de passe + token GitHub) est effacée et doit être re-saisie.

Chiffrement du token GitHub

Le token n'est jamais stocké ni affiché en clair. À chaque changement de mot de passe, il est re-chiffré automatiquement avec le nouveau dérivé SHA-256. Si aucun token n'est configuré, le changement de mot de passe fonctionne normalement sans aucune erreur.

clé = sha256(motDePasse).slice(0,16)
tokenChiffré = XOR(tokenBrut, clé) // opération bit à bit, répétition cyclique de la clé

Verrouiller l'application

Profil 🌿 → bas de page → bouton 🔒 Verrouiller. La fermeture de l'onglet ou du navigateur verrouille aussi automatiquement.

Changer le mot de passe

Profil → onglet 🔒 Sécurité → Changer le mot de passe. L'ancien mot de passe est requis. Si un token GitHub est configuré, il est re-chiffré automatiquement.

Onglet Collection

La Collection est l'écran central de l'application. Elle donne accès à tous les sujets sous deux formes : une vue liste avec cartes détaillées, et une vue Emplacements pour positionner vos sujets sur un plan interactif.

Chips statistiques

En haut de la liste : total de sujets, nombre en pot 🪴, en pleine terre 🌳, dehors ☀ (si > 0), urgent à arroser 💧 (si > 0). Ces compteurs se mettent à jour en temps réel.

Recherche & filtres

La barre de recherche filtre simultanément sur le nom vernaculaire, l'espèce et la variété. Les sélecteurs filtrent par mode de culture (pot / pleine terre) et par statut sanitaire.

Mode sélection multiple

Touchez le bouton ☑ en haut à droite pour entrer en mode sélection. Touchez les cartes à inclure, puis Appliquer une action pour appliquer à tous en une fois : fertilisation avec profil NPK, arrosage, traitement, hivernage/sortie collectif.

Ajouter un sujet

Bouton ＋ en haut à droite → formulaire complet. L'espèce se remplit avec suggestion GBIF temps réel. Les champs changent selon le mode de culture choisi (pot / pleine terre). Bouton ✕ en haut à droite de la modale pour annuler sans enregistrer.

Audit trail automatique

À chaque enregistrement de fiche, l'application compare 19 champs (espèce, variété, porte-greffe, statut, emplacement, dimensions, dates des derniers soins…). Chaque différence génère une entrée de modification dans le journal, visible dans le calendrier sous le type 📝.

Scanner le passeport phytosanitaire UE

Dans la fiche, bouton 📷 Scanner. Deux modes : vidéo en temps réel (Chrome/Edge) ou import d'image. Le code-barres standard UE est lu et les zones A (espèce), B (variété), C (porte-greffe/pays), D (agrément producteur) et E (n° lot) sont appliquées directement aux champs correspondants.

Suggestions variétés & espèces — GCVC / UCR + GBIF

La base de données des cultivars provient de la Givaudan Citrus Variety Collection (GCVC) d'UC Riverside (citrusvariety.ucr.edu ↗), l'une des collections citricoles les plus vastes au monde (~1 100 cultivars/espèces). Elle couvre 41 espèces et ~280 cultivars référencés.

Base GCVC intégrée (41 espèces)
C. × sinensis (navels, Valencia, blood), C. × clementina (Nules, Fina, Marisol…), C. unshiu (satsumas), C. reticulata (Gold Nugget, Tango, Fortune…), C. × limon (Eureka, Primofiori, Femminello…), Fortunella spp., Microcitrus australasica (finger limes), et tous les papedas, cédrats, pamplemousses, limes et exotiques.

En saisissant dans le champ Variété, les suggestions GCVC s'affichent en premier avec le badge GCVC. Si la variété n'est pas dans la base locale, une requête GBIF (species/suggest) est envoyée automatiquement après 350 ms (badge GBIF). Un lien GCVC/UCR ↗ est visible sur les champs Espèce et Variété.

Pour que vos apports apparaissent dans les jauges du tableau de bord, chaque événement fertilisation doit avoir : un profil d'engrais sélectionné (N-P-K-Ca-Mg connus) ET un poids en grammes renseigné. Sans ces deux données, l'événement est enregistré mais non comptabilisé dans les jauges.

Diagnostic maladies & ravageurs par photo

Un bouton flottant 🔬 dans chaque fiche sujet ouvre un panneau slide-up de diagnostic phytosanitaire. Deux moteurs disponibles : heuristique local (sans clé) ou Claude Vision IA (clé Anthropic dans le Profil).

Accès & workflow

  1. Fiche sujet → bouton 🔬 flottant (coin droit, au-dessus de la barre d'action)
  2. Photo — galerie de la fiche, prise de vue directe ou import fichier
  3. Contexte — zone (feuille/rameau/fruit/tronc/racine) + symptôme principal (6 options)
  4. Résultat — condition, confiance, sévérité, traitement, hypothèses alternatives

19 conditions diagnostiquées

CatégorieConditions
🐛 Insectes (6)Cochenille farineuse, Cochenille virgule, Mineuse, Araignée rouge, Puceron noir, Psylle asiatique
🍄 Champignons (5)Fumagine, Alternariose, Anthracnose, Phytophthora, Botrytis
🌿 Carences (5)Fer, Magnésium, Zinc, Azote, Asphyxie racinaire
⚠ Virus/Bact. (3)Tristeza CTV, HLB Greening, Chancre bactérien
Maladies réglementées
HLB, Psylle asiatique et Chancre bactérien déclenchent une alerte réglementaire. Isoler la plante et contacter le SRAL/DRAAF avant tout déplacement.

Les moteurs actifs s'exécutent en séquence : PlantNet confirme l'espèce → Plant.id diagnostique → Claude enrichit → heuristique local sert de filet. Des badges colorés dans le résultat indiquent quels moteurs ont tourné.

Sélection multi-zones

L'étape 2 du diagnostic permet de sélectionner plusieurs zones simultanément (ex : Feuille + Tronc). Le chip 🌿 Plante entière sélectionne toutes les zones d'un coup. Un badge compteur indique le nombre de zones actives. Lorsque la zone Feuille est sélectionnée, un sélecteur Âge des feuilles (jeunes / vieilles / toutes) s'affiche pour affiner la discrimination entre carences Fe et Mg.

Actions depuis le résultat

  • 💊 Créer un traitement — ouvre le formulaire événement pré-rempli
  • 📋 Mettre en vigilance — change le statut sanitaire
  • 📎 Sauvegarder comme observation — crée une entrée dans le journal avec badge 🔬

Historique des diagnostics

La fiche sujet affiche une section 🔬 Historique diagnostics regroupant toutes les observations issues du module, avec date, source (IA/local), score de confiance et traitement associé.

Fiabilité du moteur heuristique

Le moteur heuristique local utilise un système de portes d'exclusion strictes (hard gates) : chaque condition ne peut être proposée que si la zone ET le symptôme correspondent précisément à sa signature clinique. Les virus et bactéries réglementés (HLB, Tristeza, Chancre bactérien) sont plafonnés à 30 % de confiance maximum et toujours accompagnés d'un avertissement explicite indiquant qu'un test de laboratoire agréé est obligatoire.

Sources scientifiques intégrées

Chaque résultat de diagnostic affiche des liens vers les fiches officielles des institutions de référence : EPPO Global Database, EFSA, ANSES, CABI Compendium, UCR-CVC et USDA APHIS. Ces liens permettent de vérifier le diagnostic et de consulter des ressources validées scientifiquement avant toute décision phytosanitaire.

Configuration de la clé Claude

Profil 🌿 → section 🔬 Diagnostic IA → saisir la clé sk-ant-… + mot de passe de l'app (pour chiffrement XOR identique au token GitHub). La clé est stockée localement, jamais transmise à un serveur tiers. Anthropic ne conserve pas les images analysées.

Enregistrer la reprise après coup

Le bouton [Enregistrer la reprise] visible dans le journal sur chaque greffage sans reprise renseignée ouvre un modal dédié. La mise à jour modifie l'événement existant sans créer de doublon.

Registre des greffages (fiche sujet)

La fiche sujet affiche une section 🌿 Registre des greffages après l'historique hivernage, regroupant tous les lots avec leur taux de reprise, date de disponibilité et statistiques agrégées (taux moyen, stock en cours d'élevage).

Tableau de bord — alertes greffage

Le tableau de bord affiche automatiquement deux types d'alertes vertes :

  • ⏱ Reprise à évaluer — lot sans taux de reprise renseigné depuis plus de 21 jours
  • 🌿✓ Disponibilité imminente — plants disponibles dans moins de 28 jours (reprise déjà enregistrée)
Astuce cohortes multi-sujets
Un même lot (même lotId) peut couvrir plusieurs sujets (5 porte-greffes différents, 10 greffons chacun). Chaque sujet a son propre événement greffage avec le même lotId, ce qui permet de filtrer et regrouper par lot dans le journal.

Onglet Calendrier

L'onglet Calendrier centralise tous les événements de la collection dans une interface à 4 sous-onglets : vue mensuelle, historique fertilisation, profils d'engrais, amendements.

Vue Calendrier mensuelle

Chaque jour avec événement affiche des points colorés : la couleur indique le type d'événement (ou le profil NPK pour les fertilisations). Les jours avec rentrée hivernage ont une bordure ❄ ardoise, les sorties ont une bordure ☀ verte.

Toucher un jour déroule un panneau de détail sous la grille groupé par sujet, avec tous les événements et les modifications de fiche en accordéon dépliable. La vue scrolle automatiquement vers le panneau. Toucher le titre du sujet dans le panneau ouvre directement sa fiche.

Filtres masquables

La barre de filtres permet d'afficher/masquer chaque type individuellement ou de tout afficher / tout masquer d'un coup. Les types masqués restent visibles en transparence dans le panneau de détail, avec la mention (masqué).

Navigation mensuelle

Flèches ← → pour changer de mois. Le mois change sans réinitialiser le jour sélectionné si celui-ci existe dans le nouveau mois.

Sous-onglet 🌱 Fertilisation

Liste chronologique de tous les événements fertilisation, avec badges NPK colorés, poids apporté et amendement associé. Filtrable par sujet.

Sous-onglets ⚗ Engrais et 🪨 Amendements

CRUD complet pour les profils d'engrais (N, P, K, Ca, Mg en %) et les profils d'amendements. 6 profils d'engrais et 8 amendements sont pré-configurés. Chaque profil d'engrais a une couleur distinctive qui s'affiche sur la grille du calendrier.

Vergers — Plan interactif

La vue Vergers combine une carte IGN compacte pour gérer vos parcelles et un plan 2D SVG interactif pour positionner chaque arbre. L'interface propose deux modes : Vue (lecture seule + téléchargement) et Édition (outils complets).

Architecture de l'écran

  1. Carte IGN (360 px) — navigation, sélection cadastrale, GPS
  2. Onglets de parcelles — une tab par parcelle, changement avec sauvegarde automatique
  3. Plan 2D SVG — forme de la parcelle + arbres positionnés
  4. Barre de vue (compteur arbres + boutons Modifier / Télécharger) ou Barre d'édition (palette + outils)

Positionnement de la carte — stratégie GPS-first

La carte démarre toujours depuis votre position GPS réelle, jamais depuis les données cadastrales stockées. Cela évite qu'une boundary corrompue ne fasse retourner de mauvaises coordonnées à tous les appels IGN suivants.

Sélectionner une parcelle cadastrale

Pourquoi le réticule ?
Sur mobile, les coordonnées d'un tap peuvent être décalées de plusieurs kilomètres si la page a scrollé depuis l'init de la carte. Le réticule fixe + getCenter() sont toujours exacts.
  1. Appuyer sur 📍 GPS pour centrer.
  2. Appuyer une première fois sur 🏚 Cadastre — réticule ⊕ affiché, label « Centrez sur votre parcelle → 🏚 ».
  3. Panoramique jusqu'à placer la parcelle voulue sous le réticule.
  4. Appuyer à nouveau sur 🏚 Cadastre — identification via geo.api.gouv.fr (commune INSEE) + Nominatim (rue), puis requête IGN filtrée par INSEE.
  5. Si plusieurs parcelles : naviguer avec ‹ › ou taper le numéro visible (ex. 0128 ou BA 0128) dans le champ de recherche — la parcelle sous le réticule est pré-sélectionnée avec le badge ⊕ sous le réticule.
  6. Choisir 🌳 Créer un nouveau verger avec cette parcelle (nom pré-rempli) ou ✓ Appliquer à l'onglet actif.

Informations affichées

  • Disque SVG — illumination calculée par cosinus de la phase, portion visible gauche/droite selon croissant/décroissant
  • Pourcentage illuminé — formule : (1 − cos(phase × 2π)) / 2

Précision scientifique
La formule synodique donne une précision de ± 1 heure sur la date des phases principales. Pour les interventions critiques (greffage, taille de rajeunissement), croisez avec un éphéméride de référence.

Historique des températures vécues

Pour chaque sujet éligible (pleine terre, ou pot à l'extérieur), les températures minimales et maximales journalières sont enregistrées automatiquement à chaque chargement de la météo.

Éligibilité des sujets

Un sujet est enregistré si son emplacement est l'un de :

extérieur · jardin · terrasse · pied de mur exposé S/SE/SO · plein champ

Les plantes en intérieur, orangerie, serre ou véranda sont exclues. Si un sujet rentre en hivernage, l'enregistrement est suspendu mais l'historique existant est intégralement conservé. Dès que la plante ressort, les nouveaux relevés s'ajoutent à la suite.

Contenu de l'historique (fiche sujet)

  • Record froid absolu — température minimale jamais enregistrée, avec la date
  • Record chaud absolu — température maximale, avec la date
  • Nombre de jours enregistrés et ville météo de référence
  • Graphique des 30 derniers jours — barre min/max par jour, dégradé bleu→rouge, badge ❄ gel si T°min ≤ 2 °C
  • Liste complète — accordéon avec tous les relevés (365 max)
Utilité botanique
Connaître la température minimale réellement vécue par un sujet permet de valider sa rusticité et de calibrer les seuils d'hivernage pour les saisons suivantes. Un Citrus × junos ayant supporté −8 °C en pleine terre sans dommages est une donnée irremplaçable.

Évapotranspiration (ETP) — mode expert

Le mode ETP calcule l'évapotranspiration potentielle journalière selon la formule de Hargreaves-Samani (FAO-56), disponible à partir des données Open-Meteo déjà chargées.

Formule

ETP (mm/j) = 0.0023 × Ra × (Tmax − Tmin)^0.5 × (Tmoy + 17.8)

Ra (rayonnement extraterrestre) est calculé localement par formule analytique à partir de la latitude et du jour de l'année — aucune donnée supplémentaire n'est requise.

Coefficient cultural Kc

Un coefficient Kc mensuel est appliqué par espèce (source FAO-56 + INRAE 2023) pour obtenir l'ETc (évapotranspiration culturale) : ETc = ETP × Kc × facteur surface. Le facteur surface est estimé depuis la taille du pot.

Activation

Profil 🌿 → onglet 🌿 Profil → toggle Mode ETP expert. Quand actif, les jauges d'arrosage affichent l'ETc en mm/j en sous-label et un panneau ETP s'affiche sur le tableau de bord.

Offline total
Le calcul ETP est entièrement local — pas d'API supplémentaire. Les seules données requises (Tmax, Tmin, latitude) sont déjà disponibles via Open-Meteo ou la configuration de profil.

Alertes DAR

Le registre calcule automatiquement si le délai avant récolte a été respecté en croisant la date du traitement avec les récoltes enregistrées sur le même sujet. Les lignes non conformes sont colorées en rouge dans le PDF.

Configuration des sondes sol connectées (HTTP/JSON). Endpoint, token, intervalle de polling. Alertes humidité basse/haute configurables par sonde. Les lectures remplacent le calcul heuristique d'arrosage si les données sont fraîches (< 2h).

Données, export & sauvegarde

Toutes les données sont stockées localement dans le localStorage du navigateur. L'export JSON est votre seule sauvegarde externe — faites-le régulièrement.

Export JSON

Profil 🌿 → ⬇ Exporter JSON. Télécharge un fichier agrumes-collection-YYYY-MM-DD.json contenant l'intégralité de la collection : tous les sujets, événements, historiques de température, coordonnées GPS, photos (URLs). Conservez ce fichier en lieu sûr (Drive, NAS, e-mail à soi-même…).

Import JSON

Profil 🌿 → ⬆ Importer JSON. Remplace la collection courante après confirmation. Utilisez-le pour restaurer depuis une sauvegarde ou transférer vers un autre appareil.

Import — attention
L'import est destructif : la collection courante est effacée et remplacée par le contenu du fichier JSON. Exportez avant d'importer.

Limites du localStorage

Le localStorage est limité à environ 5 Mo par domaine. Avec les photos hébergées sur GitHub (seules les URLs sont stockées en local), vous pouvez gérer plusieurs centaines de sujets. Si vous stockez des photos en base64 directement (sans GitHub), les données volumineuses peuvent saturer l'espace rapidement.

Transfert vers un autre appareil

  1. Sur l'appareil source : Profil → ⬇ Exporter JSON
  2. Transférez le fichier .json vers le nouvel appareil (e-mail, Drive…)
  3. Copiez index.html sur le nouvel appareil
  4. Ouvrez index.html, complétez l'assistant de configuration
  5. Profil → ⬆ Importer JSON → sélectionnez le fichier exporté

Hors-ligne

L'application fonctionne entièrement hors-ligne sauf : module météo (API Open-Meteo), fonds de plan Verger (tuiles IGN/OSM), recherche GBIF et upload de photos GitHub. Toutes les fonctions de saisie, consultation, calcul et export fonctionnent sans connexion.

Données de collection
Les données enregistrées (noms des plantes, descriptions d'événements, notes) restent dans la langue dans laquelle elles ont été saisies. Seule l'interface change de langue — vos données ne sont jamais modifiées lors d'un changement de langue.
Architecture technique
Le système i18n utilise un objet LANGS contenant les traductions pour les 5 langues. La fonction getLd() retourne l'objet de la langue active. initI18n() reconstruit les constantes dynamiques (ET, STATUS, LO_POT…) à chaque chargement via buildET() et buildSTATUS(). setLang(l) sauvegarde la langue dans la config et recharge la page.
